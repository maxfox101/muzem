# Скрипты БД hero_memorial

## Кодировка в консоли Windows

Если при запуске скриптов через `psql` в cmd сообщения отображаются «кракозябрами», переключите консоль в UTF-8:

```bat
chcp 65001
```

Либо запускайте готовые bat-файлы — они делают это автоматически.

## Файлы

| Файл | Назначение |
|------|------------|
| **schema.sql** | Создание таблиц и начальные данные. Повторный запуск безопасен (CREATE IF NOT EXISTS, ON CONFLICT). |
| **001_migrate_existing.sql** | Для уже существующей БД: support_contacts, колонка users.phone, последовательности. |
| **002_fix_dictionaries_utf8_safe.sql** | Исправление «кракозябр» в справочниках (UPDATE по id, без DELETE). Можно запускать при наличии героев/заявок. |
| **fix-dictionaries-utf8.sql** | Полная перезаливка справочников (DELETE + INSERT). Только для пустой БД или без героев/заявок. |

## Что значит вывод psql

- **ПРИМЕЧАНИЕ: отношение "…" уже существует, пропускаем** — нормально при повторном запуске schema.sql.
- **INSERT 0 0** — вставлено 0 строк (все значения уже есть по ON CONFLICT), это нормально.
- **INSERT 0 18** (и т.п.) — вставлено указанное число строк.

## Исправление выпадающих списков на сайте

Если в форме звания/населённые пункты/места службы отображаются неверно:

1. Запустите **run_fix_utf8_safe.bat** или вручную:
   ```bat
   chcp 65001
   "C:\Program Files\PostgreSQL\18\bin\psql.exe" -U postgres -d hero_memorial -f "путь\к\backend\database\002_fix_dictionaries_utf8_safe.sql"
   ```
2. Либо в `backend\.env` добавьте `FIX_DICTIONARIES_UTF8=1`, перезапустите бэкенд, затем уберите переменную.

После этого обновите страницу в браузере.
